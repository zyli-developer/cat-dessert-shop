# 端到端通关流程测试

> 以真实玩家视角覆盖从启动到结算到排行榜的完整主流程。
> 手工执行为主，建议在抖音开发者工具 + 真机各跑一遍。

---

## 执行前准备

- Server：`npm run start:dev`（或指向测试环境）
- 客户端：抖音开发者工具打开 `client/build/bytedance-mini-game`
- 测试账号：至少 3 个（用于排行榜对比）
- 清空账号进度（重置内存库 / 测试账号 progress 字段）

---

## 主线路径

### TC-E2E-001  新玩家首次通关全流程
- **优先级**：P0
- **步骤**：
  1. 冷启动 → Loading 进度条 → Home
  2. 点击"开始"进入关卡 1
  3. 按教学提示掉落、合成、送餐
  4. 完成所有订单 → 结算弹窗出现
  5. 点击"下一关"进入关卡 2
- **预期**：
  - 每步 UI 正常、无报错
  - 结算页显示分数、星级、用时、奖励猫币
  - 关卡 1 在 Home 关卡选择页变为已通关并亮星
  - 后端 `GET /user/progress` 返回对应成绩

### TC-E2E-002  关卡失败流程
- **优先级**：P0
- **步骤**：持续掉落直到触发 Overflow
- **预期**：
  - Fail 弹窗出现
  - 选项：重试 / 返回主页 / 看广告复活（若接入）
  - 重试按钮重置关卡状态（分数、订单、容器、倒计时全清）

### TC-E2E-003  通关后刷新排行榜
- **优先级**：P0
- **步骤**：完成关卡 3 上传分数 → 进入排行榜页 → 切换到关卡 3
- **预期**：自己的成绩出现在列表，排名与其他账号的分数一致

### TC-E2E-004  连续通关 3 关
- **优先级**：P1
- **步骤**：依次通关 1 → 2 → 3（不退出）
- **预期**：场景切换无内存泄漏（帧率稳定）、分数正确累计到 GameState.totalScore

### TC-E2E-005  退出再回到断点
- **优先级**：P1
- **步骤**：关卡 4 打到一半关闭客户端 → 重进
- **预期**：回到 Home 关卡选择页；关卡 4 未通关状态；不恢复中途存档（当前版本不支持中途存档）

### TC-E2E-006  星级评价上限
- **优先级**：P1
- **步骤**：某关先拿 1 星，再重打拿 3 星
- **预期**：Home 显示最高 3 星

### TC-E2E-007  全关卡通关
- **优先级**：P2
- **步骤**：依次打通 10 关
- **预期**：
  - 各关配置按 `levels.json` 难度递增
  - 最终关通关后有结局彩蛋 / 总结页（如设计要求）

## 辅助流程

### TC-E2E-008  Home 页入口
- **优先级**：P1
- **检查点**：关卡选择、排行榜、设置、分享按钮均可正常进入

### TC-E2E-009  设置页
- **优先级**：P2
- **检查点**：BGM / SFX 开关状态持久化（重启后仍生效）

### TC-E2E-010  分享流程
- **优先级**：P1
- **步骤**：结算页点击分享
- **预期**：触发 `tt.shareAppMessage`；回调成功/取消均不崩溃

## 性能 / 体验断言

| 维度 | 目标 |
|------|------|
| 冷启动耗时 | ≤ 3 秒至 Loading；≤ 8 秒至 Home |
| 游戏内帧率 | 中端机 ≥ 45 FPS，低端机 ≥ 30 FPS |
| 场景切换 | < 1.5 秒 |
| 内存峰值 | < 200MB |
| 连续 10 关 | 无崩溃、无明显卡顿增长 |
