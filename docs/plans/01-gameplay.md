# 核心玩法

> 所属项目：猫咪甜品店
> 总文档：[README.md](README.md)

---

## 1. 合成链（8 级甜品）

```
Lv1 饼干 → Lv2 曲奇 → Lv3 泡芙 → Lv4 铜锣烧 → Lv5 鲷鱼烧 → Lv6 瑞士卷 → Lv7 蛋糕卷 → Lv8 奶油蛋糕
```

- 两个相同等级的甜品碰撞 → 合成为下一等级
- 每级元素体积递增（Lv1 最小，Lv8 最大），尺寸参数如下：

| 等级 | 甜品 | 直径（px） | 占容器宽度比 |
|------|------|-----------|------------|
| Lv1 | 饼干 | 40 | 10% |
| Lv2 | 曲奇 | 56 | 14% |
| Lv3 | 泡芙 | 72 | 18% |
| Lv4 | 铜锣烧 | 92 | 23% |
| Lv5 | 鲷鱼烧 | 112 | 28% |
| Lv6 | 瑞士卷 | 136 | 34% |
| Lv7 | 蛋糕卷 | 160 | 40% |
| Lv8 | 奶油蛋糕 | 188 | 47% |

> 增长规律：约每级 × 1.25 倍。容器宽 400px，两个 Lv8 并排（376px）可放入。

- 两个 Lv8 碰撞 → 消除并获得 **50 金币**

---

## 2. 投放机制

- 玩家左右滑动选择投放位置，松手自动掉落
- 顶部显示当前元素，右上角预览 NEXT 元素
- 掉落元素范围动态调整：
  - 回合 1-3：Lv1-2
  - 回合 4-7：Lv1-3
  - 回合 8+：Lv1-4

---

## 3. 物理系统

- 使用 Cocos Creator 内置 Box2D 物理引擎
- 甜品为圆形刚体，有重力和碰撞
- **容器尺寸**：400 × 600 px（设计分辨率 720 × 1280）
- **物理参数**（初始值，playtest 后微调）：

| 参数 | 值 | 说明 |
|------|------|------|
| 重力 | -960 px/s² | Box2D 默认约 -10 m/s²，按 96px/m 换算 |
| 弹性系数 | 0.3 | 轻微弹跳，不至于乱飞 |
| 摩擦系数 | 0.5 | 中等摩擦，甜品堆叠稳定 |
| 线性阻尼 | 1.0 | 防止甜品长时间滑动 |
| 投放冷却 | 0.8 秒 | 松手掉落后 0.8 秒才能投放下一个 |

- 合成反馈（Juice）：

| 事件 | 视觉 | 音效 |
|------|------|------|
| 普通合成 | 短暂缩放弹跳 + 小粒子爆发 | 清脆叮咚 |
| 连锁合成 | 连击数字（x2, x3...）+ 屏幕轻震 | 叮咚音效 pitch 递增 |
| Lv8 消除 | 全屏闪白 + 大粒子 + 金币飞散 | 华丽庆祝音效 |
| 服务顾客 | 甜品飞向猫咪 + 猫咪开心表情 | whoosh + 猫叫 |

> 连锁合成不额外加分，只提供视觉/音效递增反馈。

---

## 4. 送餐机制

- **合成即飞（设计 A）**：合成出目标甜品的瞬间，该甜品立刻飞向顾客，不留在容器中
- 如顾客需要 Lv4 x2，合成出第 1 个 Lv4 时立刻飞走，气泡显示"1/2"进度，合成出第 2 个时再飞走完成需求
- 飞行动画约 0.5 秒，弧线轨迹 + 缩放效果
- 顾客收到全部需求后播放满意表情动画
- 非目标甜品不会飞走，留在容器中参与后续合成或 Lv8 消除

---

## 5. 分数计算规则

采用**指数递增**计分，高等级合成给予高额分数奖励，强化正反馈：

| 得分事件 | 分数 |
|---------|------|
| 合成 Lv2 | +10 |
| 合成 Lv3 | +20 |
| 合成 Lv4 | +40 |
| 合成 Lv5 | +80 |
| 合成 Lv6 | +160 |
| 合成 Lv7 | +320 |
| 合成 Lv8 | +640 |
| Lv8 消除（两个 Lv8 碰撞） | +1000 |
| 服务顾客 | +50/人 |

> 公式规律：合成 LvN 得分 = `10 × 2^(N-2)`（N≥2），Lv8 消除为特殊奖励。

---

## 6. 溢出处理

- 容器顶部下方约 10% 位置设警戒虚线
- 食物超过警戒线后 **5 秒倒计时**
- 倒计时内降回去则继续，否则回合失败
