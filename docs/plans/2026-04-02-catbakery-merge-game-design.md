# 猫咪甜品店 — 合成消消乐游戏设计文档

> 日期：2026-04-02
> 平台：抖音小游戏
> 引擎：Cocos Creator 3.x + TypeScript
> 后端：Node.js (NestJS) + MongoDB

---

## 一、游戏概述

一款竖屏合成休闲小游戏。玩家在甜品店的玻璃容器中投放甜品元素，相同元素碰撞合成为更高级甜品（类似合成大西瓜）。左侧有猫咪顾客点单，合成出目标甜品后自动送给顾客。服务完当前回合所有顾客即通关。

**核心卖点**：合成物理玩法 + 餐厅经营目标 + 可爱猫咪角色

---

## 二、核心玩法

### 2.1 合成链（8 级甜品）

```
Lv1 饼干 → Lv2 曲奇 → Lv3 泡芙 → Lv4 铜锣烧 → Lv5 鲷鱼烧 → Lv6 瑞士卷 → Lv7 蛋糕卷 → Lv8 奶油蛋糕
```

- 两个相同等级的甜品碰撞 → 合成为下一等级
- 每级元素体积递增（Lv1 最小，Lv8 最大）
- 两个 Lv8 碰撞 → 消除并获得 **50 金币**

### 2.2 投放机制

- 玩家左右滑动选择投放位置，松手自动掉落
- 顶部显示当前元素，右上角预览 NEXT 元素
- 掉落元素范围动态调整：
  - 回合 1-3：Lv1-2
  - 回合 4-7：Lv1-3
  - 回合 8+：Lv1-4

### 2.3 物理系统

- 使用 Cocos Creator 内置 Box2D 物理引擎
- 甜品为圆形/椭圆形刚体，有重力和碰撞
- 合成时播放合并动画 + 粒子特效

### 2.4 送餐机制

- 合成出目标食物瞬间，该食物自动飞向左侧顾客
- 飞行动画约 0.5 秒，弧线轨迹 + 缩放效果
- 顾客收到后播放满意表情动画

### 2.5 溢出处理

- 容器顶部下方约 10% 位置设警戒虚线
- 食物超过警戒线后 **5 秒倒计时**
- 倒计时内降回去则继续，否则回合失败

## 三、顾客与关卡系统

### 3.1 顾客系统

- 左侧显示当前顾客猫咪，头顶气泡显示需求
- 顾客有不同外观（衣服/颜色/配饰随机组合）
- 送餐完成 → 满意动画 → 离开 → 下一位入场

### 3.2 难度渐进策略

| 回合 | 顾客数 | 需求类型 | 最高需求等级 | 示例 |
|------|--------|---------|------------|------|
| 1-3 | 1-2 个 | 单一需求 x1 | Lv3 | 1个泡芙 |
| 4-7 | 2-3 个 | 单一需求 x1-2 | Lv4 | 2个铜锣烧 |
| 8-12 | 3-4 个 | 多数量需求 | Lv5 | 3个鲷鱼烧 |
| 13-18 | 3-5 个 | 组合需求 | Lv6 | 1个瑞士卷 + 2个泡芙 |
| 19+ | 4-6 个 | 组合需求 | Lv7 | 1个蛋糕卷 + 2个鲷鱼烧 + 1个曲奇 |

### 3.3 通关与失败

- **通关条件**：服务完当前回合所有顾客
- **失败条件**：容器溢出警戒线超过 5 秒
- **通关结算**：得分、合成次数、用时 → 星级评价（1-3 星）
- **失败选项**：重新挑战 / 看广告复活（清除容器上半部分元素）

### 3.4 关卡配置（JSON 驱动）

```json
{
  "round": 5,
  "customers": [
    { "demands": [{ "level": 3, "count": 2 }] },
    { "demands": [{ "level": 4, "count": 1 }] }
  ],
  "dropRange": [1, 3]
}
```

## 四、UI 布局

### 4.1 页面一：加载页（自动登录）

```
┌──────────────────────────┐
│                            │
│                            │
│        🐱 猫咪甜品店       │
│                            │
```

- 自动调用抖音登录 API，无需用户操作
- 登录成功后自动跳转主页

### 4.2 页面二：主页（欢迎页）

```
┌──────────────────────────┐
│                            │
│  [🏆]              [⚙️]   │  ← 左悬浮：排行榜  右悬浮：设置
│  [🛒]                      │  ← 左悬浮：猫币商店
│                            │
│      🐱 猫咪甜品店         │  ← 甜品店门面背景
│                            │
│       ⭐⭐☆  第 5 关       │  ← 当前关卡信息+星级
│                            │
│     [◀ 切换关卡 ▶]         │  ← 关卡切换按钮
│                            │
│      [ 🎮 开始游戏 ]       │  ← 大按钮
│                            │
│  🐱💰 128                  │  ← 猫币余额
└──────────────────────────┘
```

- 左侧悬浮按钮：🏆排行榜、🛒猫币商店（后续开放）
- 右侧悬浮按钮：⚙️设置（音效/BGM 开关）
- 中间：甜品店门面背景 + 当前关卡号 + 星级
- 左右箭头切换已解锁关卡
- 底部显示猫币余额

### 4.3 页面三：游戏内界面（竖屏 9:16）

```
┌──────────────────────────┐
│ ⏸  💰15   [回合 5]  [NEXT]│  ← 第一行：暂停/金币/回合/预览
│ [🔨15] [🔀20] [📺+10]    │  ← 第二行：道具/看广告
│                            │
│  😺            ┌──────┐   │
│ 猫咪           │      │   │
│ 顾客   💬      │ 容器 │   │
│ [需求气泡]     │      │   │
│                │ 🍪🍩 │   │
│                │🧁🍞🍰│   │
│                └──────┘   │
│                            │
└──────────────────────────┘
```

### 4.4 界面元素说明

- **第一行 HUD**：⏸暂停icon（最左）| 💰金币 | 回合数 | NEXT预览（最右）
- **第二行道具栏**：🔨锤子(15) | 🔀洗牌(20) | 📺看广告(+10金币)
- **左侧顾客区**：猫咪角色 + 需求气泡（食物图标 x 数量）
- **中部容器**：玻璃容器，带警戒线虚线
- **猫币不在游戏内显示**，仅在通关结算和主菜单显示

### 4.5 界面流程

```
加载页(自动登录) → 主页(欢迎页) → 游戏内 → 通关结算 → 返回主页
                                    ↓
                                失败结算 → 重试/广告复活
```

### 4.6 弹窗

- **暂停弹窗**：继续 / 重新开始 / 返回主菜单
- **通关弹窗**：星级评价 + 分数 + 猫币奖励 + 分享按钮 + 下一关
- **失败弹窗**：重试 + 看广告复活
- **排行榜弹窗**：好友排名 + 我的排名
- **设置弹窗**：BGM 开关 + 音效开关

## 五、货币与道具系统

### 5.1 双货币体系

| 货币 | 用途 | 来源 | 特性 |
|------|------|------|------|
| **金币** 💰 | 购买道具（锤子/洗牌） | 每回合初始 15 个 + Lv8合成 +50 + 看广告 +10 | 每回合重置为 15 |
| **猫币** 🐱 | 购买家具装饰（后续开放） | 通关奖励（按星级） | 永久累积，存服务端 |

### 5.2 道具

| 道具 | 效果 | 金币价格 |
|------|------|---------|
| **锤子** 🔨 | 点击消除容器内任意一个元素 | 15 |
| **洗牌** 🔀 | 随机重排容器内所有元素位置 | 20 |

- 无使用次数限制，有金币就能用
- 道具栏后常驻广告按钮，点击看广告 +10 金币

### 5.3 通关猫币奖励

| 星级 | 猫币奖励 |
|------|---------|
| ⭐ | +5 |
| ⭐⭐ | +10 |
| ⭐⭐⭐ | +20 |

## 六、美术与音效

### 6.1 美术风格

- **卡通手绘风**：暖色调、圆润线条、手绘质感
- 甜品店背景：木质柜台、暖光、甜品展示架
- 猫咪顾客：不同衣服/颜色/配饰随机组合
- 8 级甜品各有独特外观，体积递增

### 6.2 音效

- **BGM**：甜品店轻松爵士/轻音乐
- **音效列表**：
  - 元素掉落：轻柔落地声
  - 合成成功：清脆叮咚 + 等级越高越华丽
  - 送餐飞出：whoosh 音效
  - 顾客满意：猫咪开心叫声
  - 通关：欢快庆祝音效
  - 失败：低沉失落音效
  - 道具使用：对应特效音
  - 金币获得：硬币叮当声

## 七、后端架构

### 7.1 技术栈

- **服务端**：Node.js + NestJS + TypeScript
- **数据库**：MongoDB
- **部署**：Docker 容器化

### 7.2 API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/auth/login` | POST | 抖音登录，获取/创建用户 |
| `/api/user/profile` | GET | 获取用户数据（猫币、进度、最高分） |
| `/api/user/progress` | POST | 上报回合通关结果（星级、分数、猫币） |
| `/api/rank/friends` | GET | 好友排行榜 |
| `/api/rank/global` | GET | 全局排行榜（Top 100） |

### 7.3 数据模型

```
User {
  openId        // 抖音用户标识
  nickname      // 昵称
  avatar        // 头像
  catCoins      // 猫币（永久累积）
  currentRound  // 当前回合进度
  highScore     // 最高分
  stars         // 每回合星级记录 { round: stars }
  createdAt
  updatedAt
}
```

### 7.4 排行榜

- **好友排行**：基于抖音关系链，按最高回合数排名
- **全局排行**：按最高分数排名，取 Top 100

## 八、变现与分享

### 8.1 广告变现

| 广告位 | 类型 | 触发场景 |
|--------|------|---------|
| 道具栏广告按钮 | 激励视频 | 玩家主动点击，+10 金币 |
| 复活广告 | 激励视频 | 失败后看广告，清除容器上半部分继续 |
| 通关翻倍 | 激励视频 | 通关后看广告，猫币奖励 x2 |
| 关卡间插屏 | 插屏广告 | 每 3 关通关后展示一次 |

### 8.2 分享设计

- 通关结算页：分享成绩卡片到抖音
- 卡片内容：回合数 + 星级 + 最高合成等级 + 小游戏入口码
- 支持抖音录屏 API，录制精彩合成瞬间

### 8.3 首次体验

- 前 2 关无任何广告
- 第 1 关为新手引导关（1 个顾客，需求 Lv2 x1）
