# MVP 范围与风险指标

> 所属项目：猫咪甜品店
> 总文档：[README.md](README.md)

---

## 1. MVP 范围定义

MVP（v1.0）聚焦核心合成玩法，裁剪非必要功能：

| 功能 | MVP 范围 | 说明 |
|------|---------|------|
| 关卡数量 | 10 关 | 覆盖前两个难度梯度 |
| 新手引导 | 无 | 第 1 关自然教学（1 个顾客要 Lv2 x1） |
| 猫咪顾客 | 2-3 种固定形象 | 随机出场，不做组合系统 |
| 音效 | 3 个核心音效 + 1 BGM | 合成、通关、失败 |
| 广告 | 激励视频 x4 | 道具栏 +10 金币 + 失败复活 + 主页 +10 猫币 + 通关猫币翻倍 |
| 分享 | 分享卡片 | `tt.shareAppMessage`，不做录屏 |
| 星级 | 展示 + 猫币奖励 | 预留解锁门槛数据结构 |
| 弹窗 | Prefab 动态加载 | 暂停、通关、失败、排行榜、设置 |
| 家园 | 主页预告按钮 | 灰色"家园(即将开放)"按钮，不可点击 |

## 2. 设计风险

| # | 假设 | 风险 | 验证方式 |
|---|------|------|---------|
| 1 | 合成 + 顾客目标比纯合成更有趣 | 顾客系统可能让玩家觉得"被限制"，不如自由合成爽 | 对比测试：有/无顾客需求各玩 5 分钟，问哪个更好玩 |
| 2 | 5 秒溢出倒计时合理 | 太短→挫败感，太长→无紧张感 | 观察失败率，目标：30-40% 的失败由溢出触发 |
| 3 | 金币 15 初始 + 道具定价合理 | 锤子/洗牌都 15 = 全部初始金币，玩家可能舍不得用 | 观察道具使用率，如果 <10% 说明定价过高 |
| 4 | 10 关内容量足够 MVP | 玩家可能 1-2 小时打完全部内容 | 观察通关后留存——打完 10 关后是否还会回来刷星/刷排名 |
| 5 | 猫咪主题能吸引目标用户点进来 | 抖音小游戏竞争激烈，主题可能不够差异化 | 分享卡片点击率，目标 >5% |

## 3. 成功指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 单局时长 | 前期 1-2 分钟，后期 3-8 分钟 | 自然增长，符合碎片化场景 |
| 第 1 关通关率 | ≥ 95% | 自然教学有效的验证 |
| 第 10 关通关率 | ≥ 40% | 太低说明难度曲线过陡 |
| 道具使用率 | ≥ 30% 的局使用过道具 | 通过关卡难度驱动道具使用 |
| 广告观看率 | ≥ 15% 的结算触发翻倍广告 | 激励视频的变现基础 |
| 次日留存 | ≥ 25% | 抖音小游戏均值约 15-20%，猫咪主题应高于均值 |
| 分享率 | ≥ 5% 的通关触发分享 | 自然拉新的基础 |
