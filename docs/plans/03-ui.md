│                            │

│                            │

## 3. 页面三：游戏内界面（竖屏 9:16）

```
┌──────────────────────────┐
│ ⏸  💰15   [回合 5]  [NEXT]│  ← 第一行：暂停/金币/回合/预览
│ [🔨15] [🔀15] [📺+10]    │  ← 第二行：道具/看广告
│                            │
│  😺          ┌────────┐  │
│ 猫咪         │        │  │
│ 顾客   💬    │  容器  │  │
│ [需求气泡]   │ 400x600│  │
│              │  🍪🍩  │  │
│              │ 🧁🍞🍰 │  │
│              └────────┘  │
│                            │
└──────────────────────────┘
```

### 界面元素说明

- **第一行 HUD**：⏸暂停icon（最左）| 💰金币 | 回合数 | NEXT预览（最右）
- **第二行道具栏**：🔨锤子(15) | 🔀洗牌(15) | 📺看广告(+10金币)
- **左侧顾客区**：猫咪角色 + 需求气泡（食物图标 x 数量）
- **中部容器**：玻璃容器，带警戒线虚线
- **猫币不在游戏内显示**，仅在通关结算和主菜单显示

## 5. 失败结算页

```
┌─────── 挑战失败 ──────┐
│                        │
│     💔  第 5 关        │
│     得分：320          │
│                        │
│   容器溢出，挑战失败！  │
│                        │
│  [📺 看广告复活]       │
│  清除容器上半部分元素   │
│                        │
│  [重新挑战]  [返回主页] │
└────────────────────────┘
```

- 显示当前分数（不结算星级和猫币）
- 看广告复活按钮（优先位置，最大按钮）
- 复活效果：清除容器中 Y 坐标高于容器 50% 高度的所有甜品，继续当前关卡
- 每关只能复活 1 次
- 重新挑战：从头开始当前关卡
- 返回主页：放弃本关

## 6. 界面流程

```
加载页(自动登录) → 主页(欢迎页) → 游戏内 → 通关结算 → 返回主页
                                    ↓
                                失败结算 → 重试/广告复活
```

## 7. FTUE 首次用户体验

新玩家首次打开游戏的逐步体验：

| 时间 | 玩家看到/做的 | 设计意图 |
|------|-------------|---------|
| 0-3 秒 | 加载页：猫咪 Logo + 进度条 + "正在登录中..." | 品牌认知，等待登录 |
| 3-5 秒 | 自动跳转主页：甜品店门面 + "第 1 关" + 开始按钮 | 引导点击开始 |
| 5-8 秒 | 点击开始 → 进入第 1 关游戏界面 | 无任何弹窗/教程 |
| 8-15 秒 | 看到顶部有一个甜品（Lv1）可左右滑动，松手掉落 | 自然发现投放操作 |
| 15-30 秒 | 两个 Lv1 碰撞合成 Lv2 → 叮咚音效 + 粒子 → Lv2 飞向猫咪顾客 | 理解"合成"和"送餐"机制 |
| 30-60 秒 | 继续投放合成，凑齐第 2 个顾客的 4 个 Lv2 | 理解多数量需求 |
| 60-90 秒 | 通关结算：星级 + 猫币 + 好友排名 | 理解奖励系统，点击下一关 |

> 设计原则：零文字引导，靠第 1 关的简单性（只掉 Lv1，只需合成 Lv2）让玩家自行理解全部核心机制。

## 8. 弹窗系统

- **暂停弹窗**：继续 / 重新开始 / 返回主菜单
- **通关弹窗**：星级评价 + 分数 + 猫币奖励 + 分享按钮 + 下一关
- **失败弹窗**：重试 + 看广告复活
- **排行榜弹窗**：好友排名 + 我的排名
- **设置弹窗**：BGM 开关 + 音效开关

**实现方式**：每个弹窗为独立 **Prefab**，运行时通过 `resources.load` 动态加载，关闭时销毁。便于复用和维护。
