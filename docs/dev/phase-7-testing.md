## Task 7-3：包体优化

**目标**：抖音小游戏首包 ≤ 4MB，总包 ≤ 20MB。

**优化手段**：
1. 图片压缩：PNG → PNG-8 或 WebP（Cocos Creator 构建设置中启用纹理压缩）
2. 音频压缩：MP3 码率降至 96kbps
3. 代码压缩：构建时启用 ZIP 压缩
4. 分包加载：非首屏资源放入子包
5. 删除未使用的引擎模块（3D Physics、3D Particle、Terrain 等已关闭）

**验收标准**：
- [ ] 首包 ≤ 4MB
- [ ] 总包 ≤ 20MB
- [ ] 压缩后图片质量无明显下降

## Task 7-4：性能验证

**操作**：在抖音开发者工具的性能面板中监控。

**验收标准**：
- [ ] 游戏帧率 ≥ 30fps（容器内 20+ 个甜品时）
- [ ] 内存占用 ≤ 150MB
- [ ] 场景切换时间 ≤ 1 秒
- [ ] 合成特效不导致帧率下降
- [ ] 无内存泄漏（重复进出游戏场景 10 次后内存稳定）
