# 抖音小游戏调研报告

> 调研时间：2026 年 4 月 | 目标：个人开发者上架抖音小游戏

---

## 一、平台概述

抖音小游戏是字节跳动旗下 **抖音** App 内嵌的轻量级游戏平台，用户无需下载安装，在抖音 App 内即点即玩。抖音小游戏生态自 2019 年起步，目前已非常成熟。

**核心数据：**
- 抖音日活用户超过 **6 亿**
- 小游戏通过短视频挂载 + 算法推荐获取流量
- 平台生态成熟，变现模式完善

**与 TikTok 小游戏的区别：**
- 抖音（国内版）和 TikTok（国际版）是**完全独立的两个平台**，服务器、用户、政策均不同
- 开发者门户不同：抖音使用 `developer.open-douyin.com`，TikTok 使用 `developers.tiktok.com`
- SDK 不同：抖音使用 `tt` API，TikTok 使用 `TTMinis.game` SDK
- 抖音小游戏生态更成熟，开发文档和社区资源更丰富
- 抖音支持个人开发者（需实名认证），门槛相对较低

## 二、个人开发者注册与发布流程

### 2.1 注册流程

1. 前往 [抖音开放平台](https://developer.open-douyin.com/) 注册开发者账号
2. 完成**实名认证**（个人身份证即可）
3. 创建小游戏应用

### 2.2 个人开发者认证要求

抖音小游戏支持**个人开发者**注册，门槛较 TikTok 国际版低很多。

**需要的材料：**

| 材料 | 说明 |
|------|------|
| **身份证** | 实名认证，需本人身份证正反面 |
| **手机号** | 绑定开发者账号 |
| **软件著作权（软著）** | 游戏上线前需要提供，这是抖音/微信小游戏的硬性要求 |

> **关于软著：**
> - 软著是抖音小游戏上线的**强制要求**
> - 可以自行在 [中国版权保护中心](https://www.ccopyright.com.cn/) 申请，免费但周期较长（约 2-3 个月）
> - 也可以通过代理机构加急办理（费用约 500-2000 元，1-5 个工作日）
> - 建议在开发阶段就提前申请软著，避免耽误上线

### 2.3 开发流程

1. **创建项目** — 在抖音开放平台创建小游戏应用
2. **安装开发者工具**：
   - 下载 [抖音开发者工具（IDE）](https://developer.open-douyin.com/docs/resource/zh-CN/mini-game/develop/developer-instrument/developer-instrument-update-and-download)
3. **初始化项目** — 在开发者工具中创建或导入项目
4. **开发** — 使用游戏引擎开发，导出为抖音小游戏格式
5. **调试** — 使用抖音开发者工具模拟器 + 真机预览
6. **打包** — 代码包首包不超过 **4MB**，总包不超过 **20MB**（可使用分包加载）
7. **提交审核** — 上传到开放平台，审核周期约 **1-3 天**
8. **发布上线** — 审核通过后手动发布

### 2.4 审核要求

- 代码首包不超过 4MB，总包不超过 20MB
- 需填写基本信息（名称、图标、截图、描述等）
- 内容需符合国家法律法规和抖音社区准则
- **需要软件著作权（软著）**
- 需完成实名认证

## 三、技术架构

### 3.1 运行时

抖音小游戏基于小程序运行时，使用 JavaScript/TypeScript 开发，渲染层基于 Canvas/WebGL。

### 3.2 支持的游戏引擎

| 引擎 | 支持方式 | 推荐度 |
|------|---------|--------|
| **Cocos Creator** | 原生支持，可直接导出抖音小游戏格式 | ★★★★★ 首选 |
| **Unity** | 通过 Unity 小游戏适配方案导出 | ★★★★ 适合 3D 游戏 |
| **Laya** | 原生支持抖音小游戏导出 | ★★★★ |
| **Egret** | 支持导出 | ★★★ |
| **原生 Canvas/WebGL** | 直接开发 | ★★★ 适合简单游戏 |

### 3.3 抖音小游戏 API（tt API）

抖音提供 `tt` 客户端 JavaScript API，核心能力如下：

**登录与授权：**
- `tt.login()` — 获取登录凭证（code）
- `tt.getUserInfo()` — 获取用户信息（需用户授权）

**支付（IAP）：**
- `tt.requestGamePayment()` — 使用抖币进行支付
- 需配合服务端接口创建订单

**广告（IAA）：**
- `tt.createRewardedVideoAd()` — 激励视频广告（核心变现手段）
- `tt.createInterstitialAd()` — 插屏广告
- `tt.createBannerAd()` — Banner 广告

**其他能力：**
- `tt.vibrateShort()` / `tt.vibrateLong()` — 震动反馈
- `tt.getStorage()` / `tt.setStorage()` — 本地存储
- `tt.getSystemInfo()` — 系统信息
- `tt.onShow()` / `tt.onHide()` — 生命周期
- `tt.request()` — 网络请求
- `tt.createInnerAudioContext()` — 音频播放
- `tt.shareAppMessage()` — 分享功能
- 录屏与视频分享能力

## 四、变现模式

抖音小游戏采用 **混合变现模型（Hybrid Monetization Model）**：

### 4.1 应用内广告（IAA）

| 广告形式 | 说明 | 建议场景 |
|---------|------|---------|
| **激励视频广告** | 用户主动观看 15-30 秒视频获取奖励 | 复活、奖励翻倍、解锁道具 — **核心收入来源** |
| **插屏广告** | 全屏广告，自然间隙展示 | 关卡切换、暂停恢复 |
| **Banner 广告** | 页面底部横幅广告 | 常驻展示，收益较低 |

> 激励视频广告是小游戏最主要的收入来源。

### 4.2 应用内购买（IAP）

- 使用 **抖币** 支付体系
- 用户购买虚拟道具、皮肤、去广告等
- 适合有深度付费设计的游戏

### 4.3 变现策略建议（个人开发者）

- **以广告为主**：激励视频广告门槛最低，适合个人开发者
- **免费 + 广告**模式最为稳妥
- IAP 可作为补充收入（高级皮肤、去广告等）
- 控制广告频率，避免影响用户体验

## 五、热门游戏类型分析

### 5.1 抖音上的热门品类

| 品类 | 特点 | 适合个人开发者 |
|------|------|-------------|
| **休闲益智** | 上手简单，单局短，最主流 | ★★★★★ |
| **超休闲** | 一键操作，极简玩法 | ★★★★★ |
| **合成/消除类** | 经典玩法，开发成本低，易传播 | ★★★★★ |
| **解谜/脑洞类** | 话题性强，容易社交分享 | ★★★★ |
| **放置/模拟经营** | 留存好，但开发周期较长 | ★★★ |
| **竞技/对战类** | 社交驱动，但需要服务器 | ★★ |

### 5.2 成功案例参考

| 游戏/模式 | 品类 | 爆款要素 |
|----------|------|---------|
| 合成大西瓜 | 合成类 | 玩法极简、上瘾性强、病毒传播 |
| 羊了个羊 | 消除类 | 极高难度制造话题、社交裂变 |
| 贪吃苹果蛇 | 休闲类 | 抖音热榜数周、6.5 亿+讨论量 |
| 画线火柴人 | 解谜类 | 22 亿+讨论量、创意解谜 |
| 知识就是力量 | 答题类 | 登顶 App Store 免费榜 |

### 5.3 适合抖音的游戏设计要素

- **短视频友好**：游戏画面/结果适合被录制分享为短视频
- **话题制造**：设计容易引发讨论/吐槽/挑战的环节
- **低门槛高上限**：秒上手、难精通
- **强情绪触发**：搞笑、惊喜、愤怒、成就感
- **社交对比**：排行榜、成绩卡片、PK
- **视觉冲击力**：在信息流中能吸引注意力

## 六、个人开发者选型建议

### 6.1 推荐技术栈

```
游戏引擎：Cocos Creator 3.x（首选）
开发语言：TypeScript
导出目标：抖音小游戏
开发工具：抖音开发者工具（IDE）
```

**为什么选 Cocos Creator：**
- 抖音官方支持最好，可一键导出抖音小游戏
- 免费开源
- 可一键导出多平台（抖音、微信、Web、TikTok 等）
- 学习资源丰富，中文社区活跃
- 2D/3D 都支持

### 6.2 个人开发者推荐方向

#### 最推荐（低成本高回报）

1. **合成类游戏**
   - 开发难度：★★☆☆☆
   - 传播潜力：★★★★★
   - 例：水果合成、方块合并
   - 核心：简单物理 + 合成机制 + 上瘾循环

2. **消除/三消变体**
   - 开发难度：★★★☆☆
   - 传播潜力：★★★★★
   - 例：三消、连连看变体
   - 核心：加入独特机制制造话题

3. **答题/知识测试类**
   - 开发难度：★★☆☆☆
   - 传播潜力：★★★★☆
   - 例：IQ 测试、冷知识挑战、性格测试
   - 核心：结果分享卡片驱动传播

4. **一笔画/画线解谜**
   - 开发难度：★★☆☆☆
   - 传播潜力：★★★★☆
   - 例：画线救援、一笔画完
   - 核心：创意关卡设计

#### 进阶方向

5. **放置挂机类**
   - 开发难度：★★★☆☆
   - 留存优势：★★★★★
   - 适合长线运营

6. **弹射/物理类**
   - 开发难度：★★★☆☆
   - 观赏性：★★★★★
   - 物理效果适合短视频传播

## 七、包体大小限制与优化

抖音小游戏的包体限制较为严格（对比 TikTok 的 200MB）：

| 类型 | 限制 |
|------|------|
| **首包（主包）** | 不超过 **4MB** |
| **总包（含分包）** | 不超过 **20MB** |
| **远程资源** | 无限制，可通过 CDN 加载 |

### 优化策略

- **分包加载**：将非首屏资源放到分包中，按需加载
- **远程资源加载**：大型资源（音效、图片、动画）放到 CDN
- **图片压缩**：使用 TinyPNG 等工具压缩图片资源
- **代码压缩**：开启 Cocos Creator 的代码压缩和混淆
- **按需加载**：关卡资源懒加载，不要一次性加载全部资源

## 九、最佳实践

### 9.1 设计原则

1. **3 秒吸引**：加载后立即进入核心玩法
2. **竖屏单手**：适配竖屏，单手可操作
3. **单局 1-3 分钟**：适合碎片化使用
4. **中文为主**：面向国内用户，界面和内容使用中文

### 9.2 传播设计

- 利用抖音录屏能力，内置分享功能
- 设计挑战性关卡制造话题
- 生成可分享的成绩卡片
- 利用排行榜激发竞争
- 优化视频挂载时的展示效果
- 设计适合短视频传播的精彩瞬间

### 9.3 变现最佳实践

- 激励视频广告放在自然节点（复活、奖励翻倍等）
- 用户每 3-5 分钟遇到一次广告机会为宜
- 首次体验 1-2 关不要出现广告
- IAP 作为可选的高级体验（去广告、皮肤等）
